Grok Build CLI + AI-Hands

Give Grok Build hands.

AI-Hands gives Grok Build CLI a local MCP server for browser control, desktop UI automation, screenshots, vision, and OCR. Voice-Command and GrokCLI can layer on top, but the first install is Hands.

Grok Build config example

Add AI-Hands to Grok Build CLI

After installing AI-Hands, merge this shape into Grok's MCP config and adjust the path if your install folder differs. Add Voice only if you install the optional voice layer.

[mcp_servers.hands] command = 'C:\AIWander\servers\hands.exe' # optional: [mcp_servers.voice] command = 'C:\AIWander\servers\voice.exe'
Proof path

The demo should be Grok using AI-Hands.

The strongest proof is one coherent run inside Grok Build CLI where the model acts through the UI and verifies the outcome visually.

Open the target.Use AI-Hands to launch a browser or desktop app from Grok Build CLI.
Act through the UI.Click, type, scroll, and inspect state through the same interface a person would use.
Verify visually.Capture screenshots, use OCR or vision, and confirm the requested result is on screen.
Report the proof.Have Grok summarize the action, the visible evidence, and what still needs user approval.
